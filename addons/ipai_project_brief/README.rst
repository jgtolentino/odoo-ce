This module adds a structured "Project Brief" object to Odoo 18 CE,
designed for creative and agency workflows.

**Table of contents**

.. contents::
   :local:

Features
========

* New model: ``project.brief`` with:

  - Title, project, client, brand
  - Objective, target audience, key message, deliverables
  - Budget (monetary), deadline
  - Status workflow with tracking
  - Lock flag for approved briefs

* Status flow: Draft → In Review → Approved / On Hold / Cancelled
* Automatic lock of approved briefs (non-admin users cannot edit)
* Kanban, tree, and form views under Project menu
* Search filters and grouping options
* Chatter integration for activity tracking
* Demo data for quick testing
* Comprehensive test suite

Use Cases
=========

* Capture structured inputs for creative and strategic work before production
* Track project briefs through approval workflow
* Prevent accidental changes to approved briefs
* Group and filter briefs by client, project, or status

Configuration
=============

No configuration required. The module installs menus under Project.

Access Control
--------------

* **Project Users** (``project.group_project_user``): Can read, create, and edit briefs
* **Project Managers** (``project.group_project_manager``): Full access including delete
* **System Administrators**: Can unlock approved briefs

Usage
=====

1. Go to **Project → Project Briefs → Briefs**
2. Create a new brief and link it to a project and client
3. Fill in objectives, target audience, key message, and deliverables
4. Set budget and deadline
5. Use the status buttons to move the brief through review and approval
6. Once approved, the brief is locked for editing by non-admins

Workflow States
---------------

* **Draft**: Initial state, fully editable
* **In Review**: Under review, awaiting approval
* **Approved**: Final state, locked for editing
* **On Hold**: Temporarily paused
* **Cancelled**: Terminated
